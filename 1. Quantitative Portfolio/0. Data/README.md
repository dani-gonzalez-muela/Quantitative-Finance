# 0. Data — datasets, loaders & registry (single source of truth)

**Git note:** everything in `0. Data/` is gitignored EXCEPT this README — the
datasets are large and `loaders/data.py` holds API keys. This file documents
every data source so the project is understandable without the data itself.

This file is also machine-read by `_shared/paths.py` (the STORE-REGISTRY table
below). Code never hardcodes data paths; it calls:

```python
from _shared.paths import data_dir, data_file
DATA = data_dir('daily_tickers')                 # a folder
vix  = data_file('vix', 'VIXCLS.csv')            # a file inside a store
```

To physically move a store, edit **only** the path in the registry table below —
no code changes needed. Paths are written relative to the project root (the
folder holding the `.project_root` marker) and use `/` separators; the resolver
joins them onto the real root and normalises for Windows.

## What each dataset is

### WRDS (Wharton) — `wrds/`

Institutional market & fundamentals data pulled from WRDS.

- **`wrds`** — container for the four sub-projects below (`parquet/`, `full/`, `vol_overlay/`, `quantitative_momentum/`). Reorganized 2026-07-06: redundant `setup/` folder (exact duplicate of `full/data_infrastructure/`) removed → `_delete/`. The legacy loader shims (`data.py`, `wrds_data.py`) re-export `_shared.loaders_*` (see below).

> **The loaders (`0. Data/loaders/`)** are the code layer the strategies load through — `from _shared.loaders_data import fetch_historical_data` and `from _shared.loaders_wrds import load_vix, load_sp500_daily, load_fred_rates` (thin re-export modules in `_shared/` load these files at runtime). Built on these stores via `_shared.paths` (VIX ← `vix`/`VIXCLS.csv`, rates ← `fred_rates`/`fred_rates_synthetic.csv`, S&P 500 ← `wrds_parquet`/`05_sp500_crsp.parquet`). `data.py` contains the Alpaca API keys — which is why this folder stays out of git.
- **`wrds_parquet`** — cleaned parquet snapshots (~5.7 GB): CRSP security daily prices, S&P 500 constituents, CRSP market portfolios, world country returns, Treasury risk-free, CBOE VIX, global exchange rates, CCM master, index masters (files numbered `03`–`19`). This is what most long-term backtests read. Self-documented via `parquet/README.md`; built by `parquet/downloaders/` (the WRDS download + query notebooks).
- **`wrds_full`** / **`wrds_datasets_raw`** — the original raw WRDS_DATASETS download tree (CRSP, Compustat, Financial Ratios, MIDAS, Third-Party factors), including the source `.docx` manuals and the raw `.zip`s (Ratios.zip, MIDAS.zip, RatiosGICS10.zip, …). Consumers that need specific zips: bab_long_short, reit_dividend_carry, short_interest_contrarian, quality_v2, load_fred_rates. **Note:** individual zip filenames sit in `WRDS [done]/…` sub-folders — a consumer that expects a flat layout may still need per-file repointing.
- **`wrds_vol_overlay`** — WRDS parquets used only by the vol_overlay research (`financial_ratios.parquet`, `sp500_constituents.parquet`). Namespaced under `wrds/`, intentionally separate from `wrds_parquet`.
- **`quantitative_momentum_cache`** — strategy-private CRSP cache for QMom (`crsp_daily_demo.parquet`, `crsp_daily_full.parquet`). Formerly `caches/quantitative_momentum`; relocated under `wrds/`.

### Alpaca — `alpaca/`

Broker price data.

- **`daily_tickers`** — daily ETF OHLCV, one CSV per ticker (`SPY.csv`, `QQQ.csv`, … 68 tickers) in the `tickers/` sub-folder. Consumers load `data_dir('daily_tickers')/<TICKER>.csv`. Used by Daily (IBS MR, VIX MR), Long-Term multiasset backtests, and the downloaders.
- **`etf_metadata`** — `etf_metadata.csv`, one level above the tickers, plus the daily downloader.
- **`intraday_5min`** — 5-minute OHLCV, gzipped, ~44 tickers (`SPY_5min.csv.gz`, …). Used by all Intraday strategies.

### FRED (St. Louis Fed) — `fred/`

Macro & rates series pulled from fred.stlouisfed.org. Refreshers live in `fred/downloaders/` (`fetch_vix_data.py`, VIX3M source tests) alongside `Gathering_Data.docx` (the vol_overlay data-gathering write-up covering **both** FRED and Yahoo pulls).

- **`fred_macros`** — raw macro panel (~28 series): CPI, GDP, PPI/commodities (PPIACO), federal debt, income tax, FX (EUR/USD, CNY/USD), long-term govt bond rate, quarterly financial reports, consumer sentiment, natural gas, etc.
- **`fred_rates`** — FRED rate CSVs: `fred_dgs2/10/20`, `dfii10`, `t10y2y`, plus `fred_rates_synthetic`. Consumers: yield_curve_duration, bond_duration_carry, load_fred_rates.
- **`vix`** — `VIXCLS.csv` (FRED VIX close). Refresh via `fred/downloaders/fetch_vix_data.py`. Consumer: vix_etn_dual (which also needs `cboe_vix3m`).
- **`macros`** — **processed** outputs: `macros_daily/monthly/quarterly.csv`, regime signals, vol-model predictions, enriched panels, spy returns. Consumers: vix_mean_reversion, bond_trend, commodity_carry/trend, cross_asset_carry, credit_carry, pipelines.

### CBOE — `cboe/`

- **`cboe_vix3m`** — `VIX3M_History.csv` (3-month VIX, CBOE). Consumer: vix_etn_dual.

### Quiver Quant — `quiver/`

- **`congress_quiver`** — `final_data_MEGA.pkl` (~589 MB congressional-trading dataset). Consumer: Daily_Strategies/congress_momentum.

### Ken French / Fama-French — `fama_french/`

- **`fama_french`** — Ken French factor data: `ff_factors_monthly.csv` at the store root. Consumed by 8+ timing backtests and pipelines.
- **`regime_factor_rotation_cache`** — regime-rotation strategy cache (`sp500_daily.csv`, transformed monthly/quarterly panels, RV targets, diagnostics PNGs, `metadata.json`). Formerly `caches/regime_factor_rotation`. **Note:** `ff_factors_monthly.csv` now lives one level up in the `fama_french` store — a consumer requesting it *through* this cache key needs repointing to `fama_french`.

### Yahoo Finance — `yahoo/`

Market prices pulled via Yahoo (`yfinance`). Distinct from FRED's own PPI commodity index in `fred_macros`.

- **`yahoo_commodities`** — daily futures settle prices, `Date,Price` CSVs: gold `GC=F`, silver `SI=F`, copper `HG=F`, corn `ZC=F`, platinum `PL=F`, wheat `ZW=F`, soybeans `ZS=F`, coffee `KC=F`, sugar `SB=F`, cotton `CT=F`, plus the `XLU` utilities ETF.
- **`yahoo_rates`** — bond/rate series (`bonds_yahoo.csv`, `interest_rates_yahoo.csv`; e.g. `ZB=F`, `^TNX`).
- **`yahoo_sp500`** — `sp500_index.csv` (S&P 500 index level). This store points at the file itself.
- **`vol_overlay_raw`** — retired trove key; now points at `fred/downloaders/` where the vol_overlay research docs live. vol_overlay notebooks 01–03 should be repointed to `fred_macros` / `yahoo_commodities` / `wrds_vol_overlay`.

## Baskets

The 7 test baskets (US broad equity, US factors, US sectors, US bonds, commodities,
EM regional, international developed) are defined in `alpaca/downloaders/intraday_baskets.json`
(the download universe) and hardcoded per strategy in each `*_Backtest_multiasset_v2.py`
(`BASKETS` dict). They are documented with tickers in the dashboard's Overview tab (§2)
and the root README.
